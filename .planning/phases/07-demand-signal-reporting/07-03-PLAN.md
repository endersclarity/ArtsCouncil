**Report structure** (follow COMMITTEE-REPORT-TEMPLATE.md structure):

```markdown

**Report Period:** {month_name} {year}

## Executive Summary

This month, **{visitors} visitors** explored our cultural asset map, generating **{total_events} engagement events** and **{tier1_referrals} direct business referrals**.

## Key Metrics

| Metric | Value |
|--------|-------|

| Rank | Category | Filter Count |
|------|----------|-------------|
{for each category in demand_signals.top_categories, numbered}

| Rank | Venue | Detail Opens | Outbound Clicks | Score |
|------|-------|-------------|-----------------|-------|

{if zero_result_searches.length > 0:}
Visitors searched for these terms and found nothing — potential content gaps:

{else:}
No zero-result searches this period.

---

## Business Engagement Ranking

**Ranked by referral signal strength** (outbound clicks = strongest):

| Rank | Business | City | Category | Score | Primary Signal |
|------|----------|------|----------|-------|---------------|
{for each venue in venue_attribution top 10}

{if any venue has note field:}
**Note:** {venue.note}

---

## AI Concierge Insights

### Query Volume
- **Total queries:** {chat_logs.length}
- **Avg response time:** {avg_duration_ms}ms

### Intent Distribution

| Intent | Count | % |

{for each intent aggregated from chat_logs query_intents}

|------|-------|-----------|
{for each page in page_views}

## Feature Usage

| Feature | Count |

## Technical Notes

### Data Collection
- **Umami analytics:** Privacy-first, no cookies, tracks custom events via provider-agnostic wrapper
- **Supabase chat logs:** AI concierge queries and responses with session attribution
- **Session tracking:** {if session_hash data present: "session_hash property on all events enables per-visitor journey reconstruction" else: "Aggregate data only — session-level attribution available in future reports"}

### Known Measurement Gaps
1. Chatbot deep link clicks: {if chat:deeplink-click in events: "Now tracked" else: "Not yet instrumented — chatbot referral value may be undercounted"}
2. Session-level journeys: Umami Cloud API provides aggregate counts, not per-session event streams

---

*Report generated by Cultural Map analytics pipeline*
*Next report: {next_month_name} {next_year}*
```

**Implementation details:**

- Read the pipeline JSON with `fs.readFileSync` + `JSON.parse`
- Build each section as a string using template literals
- For tables: generate proper markdown table rows with `|` separators and alignment
- For percentages: calculate from totals (e.g., intent count / total queries * 100, rounded to 1 decimal)
- Format numbers: use toLocaleString() for comma separation on large numbers
- Format duration: convert avg_duration_ms to minutes (÷ 60000), round to 1 decimal
- For month names: use Date constructor with year/month to get locale string
- Handle missing data gracefully: if a section has no data (e.g., no chat logs), show "No data for this period" instead of empty table
